What ships in the box

Not a demo. A production-shaped reference.

Every capability below is implemented, tested, and documented with an ADR explaining why.

🔐

Short-lived signed tokens

PASETO v4.public, Ed25519, 5-minute user TTL, 60-second service TTL. Two distinct keypairs. No JWT, no algorithm confusion.

📋

Declarative RBAC

YAML with role inheritance and group-claim mapping. Hot-reloaded on file change. Reviewed in normal PR flow — no UI panel, no Terraform indirection.

📜

Append-only audit

Every gateway call lands in the store. SQLite default, ClickHouse opt-in. Public surface has no DELETE — append-only by design.

🔁

Replay protection

10,000-entry LRU cache keyed by token jti. Second sighting inside TTL is rejected. Lives in the gateway and only the gateway.

🧠

Verify-output meta-skill

Last subskill in every investigation. Walks the audit log, checks every claim against logged tool results, annotates unsupported claims inline.

✋

Human-in-the-loop actions

case_actions tools require human_approval=true on the claim. The harness does not mint approved tokens — by design.

🧪

Deterministic eval suite

Six YAML datasets, four scorers (tool correctness, ordering, grounding, reasoning). PR-gate runs deterministic; nightly runs against the live model.

📊

Pre-provisioned Grafana

Dashboard panels for tool counts, latency, denied-by-role, audit volume. Works against SQLite and ClickHouse out of the box.

🔌

Add a data source in <1 hour

Define the MCP tool schema. Drop a server using the shared factory. Declare in rbac.yaml. Reference from a subskill. Each step has a verify command.

Six built-in investigation scenarios

From the boring "all clear" to the messy edge cases.

Each persona is deterministic, cross-server-consistent, and covered by an end-to-end eval. Train the agent. Train your team. Both.

clean

Clean customer

The agent should not invent risk. Validates that low-signal cases come back as "no action." Tests for over-eager hallucination.

mule

Mule account

Inbound transfers from many small senders, fast pass-through. The agent must spot the layering and reference specific transactions.

sanctions_hit

Sanctions hit

Counterparty matches an OFAC-style watchlist. Tests sanctions-screening ordering and forces an action gated behind human approval.

ato

Account takeover

New device, new IP, new beneficiary. Validates device-history pivot and contextual reasoning across customer + transaction signals.

structuring

Structuring

Deposits just under the reporting threshold. The agent must aggregate across days and flag the pattern, not the individual transactions.

synthetic_id

Synthetic identity

Real SSN, fabricated name, thin file. Tests KYC + OSINT cross-referencing and forces the agent to distinguish "thin" from "synthetic."

The security review your auditor will run anyway

Every load-bearing decision has an ADR.

Seven Architecture Decision Records. One threat model with all seven trust boundaries mapped to their mitigation.

One algorithm. One path.

PASETO v4.public (Ed25519) exclusively — never JWT. Typed claims, no algorithm-confusion surface. Two distinct keypairs: one for the user token, one for service-to-service. The MCP gateway never reissues the user token; it re-signs a fresh service token with its own private key.

ADR-0002 · paseto-over-jwt.md

Replay protection by default

Every request carries a jti. The gateway holds an in-memory LRU of 10,000 entries and rejects the second sighting inside the TTL window. The cache lives in the gateway and only the gateway. No external state to compromise.

gateways/mcp/main.py

RBAC compliance teams can read

Roles in config/rbac.yaml with inheritance and group-claim mapping. Hot-reloaded on file change. PRs reviewable by a compliance officer in the normal code-review flow — no UI panel to audit, no Terraform indirection.

ADR-0003 · yaml-rbac.md

Audit by default. No DELETE.

Every gateway call lands in an audit store. SQLite default for laptop dev; ClickHouse opt-in for production scale. The module's public surface has no DELETE method. Append-only is enforced by absence, not by trigger.

ADR-0004 · sqlite-default-audit.md

Skills are repo-resident, not model-generated

Every SKILL.md is read verbatim — never templated by the model. Each is tied to a git commit hash in the audit log. The threat model names skill-spoofing as a tracked boundary.

docs/threat-model.md

No PII in traces. By construction.

OpenTelemetry attributes carry mcp.server, mcp.tool, user.role. user.email and user.sub are explicitly forbidden — the helper raises on violation.

gateways/common/otel.py
The eval surface

Score the agent on what actually matters.

Four scorers, six datasets, headless harness. PR-gate runs deterministic; nightly runs against the live model.

✅

tool_correctness

Set comparison of expected tool calls vs. what the audit log actually recorded. Did the agent call the right tools, full stop.

🔢

tool_ordering

Dependent-ordering checks. Sanctions must be screened before the narrative is drafted. Profile must be gathered before transactions are analyzed.

🎯

grounding

LLM-judge per claim. Does every factual statement in the SAR trace back to a logged tool result? Uses cache_control on the rubric for cost.

📈

reasoning

Five-dimension rubric, 1–5 each. Mean ≥ 4.0 to pass. Catches the failures the other three scorers miss — the report that's correct but unconvincing.
