# Fraud Investigation

You have been given a fraud alert to investigate. Use the **orchestrator** skill from this plugin to drive the investigation.

**Alert payload:** $ARGUMENTS

Construct the alert payload as:

```json
{
  "alert_id": "alert-<random>",
  "customer_id": "<first arg>",
  "alert_type": "<second arg, default mule_account>",
  "severity": "<third arg, default high>",
  "opened_at": "<now in ISO-8601>"
}
```

If the customer_id starts with `cust-mule-`, pass `scenario: "mule"`. Similarly for `cust-sanctions-`, `cust-ato-`, `cust-structuring-`, `cust-synthetic-`, `cust-clean-`.

Then follow the orchestrator skill's routing logic exactly. The MCP tools you need are exposed via the `fraud-investigator` MCP server registered by this plugin (prefixed by server name, e.g. `customer_data__get_customer`, `transactions__flag_velocity_anomalies`).

After the orchestrator completes, end with `/fraud-audit-trail` to show the user the audit DB rows generated by this investigation.
