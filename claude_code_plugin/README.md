# fraud-investigator (Claude Code plugin)

Install via Claude Code's `/plugin` GUI from the marketplace at the root of the
[`fc_oss`](https://github.com/amitberger02/fc_oss) repo. Once installed, you get:

- **7 skills** — orchestrator + 5 subskills + verify-output meta-skill, the same
  shape as Stefano Amoreli's Qonto fraud-investigator plugin (Code with Claude
  2026 talk).
- **An MCP server** named `fraud-investigator` that proxies stdio MCP calls
  from Claude Code through the local **PASETO + RBAC + audit** MCP gateway to
  six federated downstream MCP servers.
- **3 slash commands**: `/fraud-stack-up`, `/fraud-investigate`, `/fraud-audit-trail`.

## Install

```
/plugin marketplace add github:amitberger02/fc_oss
/plugin install fraud-investigator@fraud-copilot-oss
```

Restart Claude Code so the new MCP server gets picked up.

## Run

Bring up the local stack (from a checkout of the `fc_oss` repo):

```
/fraud-stack-up
```

That runs `make compose-up && make load-fixtures` under the hood and confirms the
full PASETO + RBAC path with a smoke curl.

Then in any Claude Code conversation:

```
/fraud-investigate cust-mule-01
```

Claude loads the orchestrator skill, routes through subskills, calls MCP tools
through the gateway. Every tool call is authenticated, RBAC-checked, and audited.

Finally:

```
/fraud-audit-trail
```

…to show the audit DB rows generated by the investigation.

## Demo personas

| customer_id | scenario | Notes |
|---|---|---|
| `cust-clean-01` | clean | Baseline. Orchestrator stops after gather-customer-profile. |
| `cust-mule-01` | mule | Triggers analyze-transactions + check-osint. |
| `cust-sanctions-01` | sanctions_hit | Triggers screen-sanctions; real OFAC-style hit. |
| `cust-ato-01` | ato | Account takeover signals. |
| `cust-structuring-01` | structuring | Velocity anomalies + smurfing pattern. |
| `cust-synthetic-01` | synthetic_id | KYC inconsistencies + shell-company OSINT. |

## Configuration (env vars on the MCP server)

All optional with demo-friendly defaults — see `.mcp.json`:

| Env var | Default |
|---|---|
| `FRAUD_MCP_GATEWAY_URL` | `http://localhost:8100` |
| `FRAUD_AUTH_GATEWAY_URL` | `http://localhost:8080` |
| `FRAUD_OIDC_URL` | `http://localhost:9000` |
| `FRAUD_OIDC_EMAIL` | `alice@example.com` |

> The proxy mints a fresh PASETO per gateway call (the gateway tracks jti replay
> in an LRU cache, so reusing the same PASETO across calls would fail).

## What this is *not*

This plugin is **not** a substitute for Claude Cowork. Stefano's Qonto demo
uses Cowork as the chat surface with inline interactive widgets — that's a
private-beta product. This plugin is the closest open-source equivalent:
same architecture, same auth/RBAC/audit guarantees, runs in Claude Code which
anyone has.

The Cowork-targeted plugin manifest still lives at the **repo root** under
`plugin/plugin.json` — that's the artifact `make register-plugin` validates
and hands to a `cowork` CLI when one is on PATH.
